Accessing to the Ensembl REST API

RESTful web services

REST (REpresentational State Transfer):

An architectural style for designing remote services that allow to access and operate on remote resources.

Constraints:

  • client-server
  • stateless
  • cacheable
  • layered
  • uniform interface (identification/manipulation/self-descriptive/hypermedia)

RESTful/GETful web services

HTTP is the usual protocol to access and manipulate the resources.

HTTP operations:

  • GET
  • PUT
  • DELETE
  • POST

"GETful" web services are "read-only" services (resources can be only read, not modified)

Ensembl REST API

An API for the programmatic access (read-only) to Ensembl resources developed in the REST-style.

Currently in beta, it will eventually supersede the traditional Perl (!) API.

It provides access to (not-exhaustive list):

  • Sequences (genomic, cDNA, CDS, and protein)
  • VEP (Variant Effect Predictor)
  • Homologies
  • Gene Trees
  • Assembly and coordinate mapping

It returns different representations/formats (JSON, XML, GFF3; ...) depending to the user preferences.

Ensembl REST API

Home: http://beta.rest.ensembl.org/
User guide: http://beta.rest.ensembl.org/documentation/user_guide
Endpoint documentation: http://beta.rest.ensembl.org/

Each functionality is represented by an "parametrized" URL called endpoint.

Required parameters (e.g., resource id):

  • part of the "URL path"
  • prefixed by : or enclosed by {} in the documentation

Optional parameters (e.g., output format):

  • specified as query string/HTTP header

Simple example

Example: Retrieve the nucleotide sequence of a specific genomic region.

Syntax: GET /sequence/region/:species/:region

Query: GET /sequence/region/human/X:1000000..1000100:1

Via console:
curl 'http://beta.rest.ensembl.org/sequence/region/human/X:1000000..1000100:1?' -H 'Content-type:text/x-fasta'

Output:

>chromosome:GRCh37:X:1000000:1000100:1
GAAACAGCTACTTGGAAGGCTGAAGCAGGAGGATTGTTTGAGTCTAGGAGTTTGAGGCTG
CAGTGAGTTATGAGCACACCACGGCACTCCAGCCTGGGAGA

Notes

  • Many functionalities are available: check the docs!

  • The service is in beta:
    Available for everyone, but...
    with a fair-usage limit (max 3 request per second)

Fair usage example

Command:
curl -v 'http://beta.rest.ensembl.org/sequence/region/human/X:1000000..1000100:1' -H 'Content-type:text/x-fasta'

How to programmatically access the Ensembl REST API from Ruby?

There is a Ruby Gem (of course)!

Gem: bio-ensembl-rest
Author: Alberto Donizetti (supervised by Francesco Strozzi)
Homepage: https://github.com/ALTree/bio-ensembl-rest
Usage: https://github.com/ALTree/bio-ensembl-rest/wiki

bio-ensembl-rest

bio-ensembl-rest maps each (logical) group of endpoints into a Ruby module whose methods are the endpoints available for the group.

API parameters are mapped to the method parameters, while errors are converted to Ruby exceptions.

The list of methods is available here

Installation? It is a public gem!

gem install bio-ensembl-rest

bio-ensembl-rest - Example

require 'bio-ensembl-rest'
include EnsemblRest

EnsemblRest.connect_db # connect to database
puts Sequence.sequence_region 'Homo sapiens', 'X:1000000..1000100:1'
# => GAAACAGCTACTTGGAAGGCTGAAGCAGGAGGATTGTTTGAGTCTAGGAGTTTGAGGCTG
#    CAGTGAGTTATGAGCACACCACGGCACTCCAGCCTGGGAGA

puts Sequence.sequence_region 'Homo sapiens', 'X:1000000..1000100:1', response: 'fasta'
# => >chromosome:GRCh37:X:1000000:1000100:1
#    GAAACAGCTACTTGGAAGGCTGAAGCAGGAGGATTGTTTGAGTCTAGGAGTTTGAGGCTG
#    CAGTGAGTTATGAGCACACCACGGCACTCCAGCCTGGGAGA
