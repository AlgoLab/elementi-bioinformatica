Ruby Basic Facts (part 2)

Defining functions

def defname < ( <, arg <=val> >∗ <, &blockarg> ) >
  <body>
end
  • defname is the name of the function (lowercase initial, may end with !, ?, =)
  • arg is the name of each argument and val its optional default value
  • &block as last argument (& included) is the block passed to the function
  • the return value is the result of the last executed statement

Defining functions

def count_letters(str, letters, ignore_case=false)
  freqs=[]
  if ignore_case
    str = str.downcase
    letters = letters.downcase
  end
  letters.each_char { |letter| freqs << str.count(letter) }
  freqs
end

a = "BananA"
l = "aNz"
count_letters(a, l)
# => [2, 0, 0]
count_letters(a, l, false)
# => [2, 0, 0]
count_letters(a, l, true)
# => [3, 2, 0]

Arguments are passed by value, but they are references!

Defining classes

class DNASequence < Object
public   # default is public. private and protected are also available
  def initialize(identifier)  # C'tor
      @identifier = identifier
      @seq = nil
  end

  attr_reader :identifier     # identifier is read-only
  attr_accessor :seq          # sequence is read-write

  def to_fasta
    ">#{@identifier}\n#{@seq}"
  end
end

Defining classes

We can add methods to existing classes:

class DNASequence < Object
  def reverse_and_complement!
    @seq.reverse!
    @seq.tr! "ACGTacgt", "TGCAtgca"
    self
  end
end

..even to "system" classes:

class File
  def count_non_empty_lines
    self.count { |line| line.strip.length>0}
  end
end

Defining iterators

class File
  def each_word
    self.each do |line|
      line.strip!
      line.tr! "'", " "
      '!.,;'.each_char { |mark| line.gsub!(mark, '') }
      line.split.each do |word|
        yield word     # Pass the control to the caller
      end
    end  
  end
end

freqs = Hash.new 0
File.open('ex-1.txt').each_word {|w| freqs[w.downcase] += 1}
puts freqs.max { |a,b| a[1] <=> b[1] }

Modules and Mixins

Modules are, essentially, classes without internal state.

Modules can be mixed-in (included) in a class to extend functionality.

module FriendlyModule
  def say_hello
    puts "Hello!"
  end
end

class DNASequence
  include FriendlyModule    # Now the methods of FriendlyModule are
                            # methods of DNASequence
end

s = DNASequence.new "identifier"
s.say_hello
# => Hello!

Useful mixins

If you are defining a class representing:

  • objects that can be compared, you should define the method <=> and include the module Comparable, which provides <, <=, ==, >=, >, and between?.

  • a collection of items, you should define the iterator each and include the module Enumerable, which provides 47 (!!) methods (such as member, find, each_with_index) for free.

Exercises (part 2)

Exercise

Problem 2a:
Write a class DNASequence representing a DNA sequence.
Such class should store (at least) two data: the identifier and the sequence itself. The sequence, if not empty, must contain only A, C, G, or T (lower and/or upper case). It should also define methods for computing:

  • the length of the sequence
  • its reverse and complement (two version: returning it as a copy and in-place)
  • the GC content (number of G or C divided by the length)
  • a string representing the DNA sequence in FASTA format (the sequence should be wrapped at a use-specified position, default: 80)

File: lab/examples/ex-2a-dnasequence.rb

Exercise

Problem 2b:
Write a class FASTAReader that reads a collection of DNA sequences from a FASTA file.
Since FASTA files are usually huge, you should never store the whole file in memory, but you should allow to process the sequences one by one. Sequences might be stored on different (consecutive) lines of different length.
Test the reader by processing the file ex-2.fa for computing:

  • the length of the longest DNA sequence
  • the identifiers of all the longest sequences
  • a sequence with the lowest GC content
  • the number of sequences containing the motif (pattern) CAGCTCCAGCTCCAGC
